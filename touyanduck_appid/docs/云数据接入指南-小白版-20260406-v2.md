## 一、整体原理

当前数据流是这样的：

```
你说"投资Agent" / "投资App"
  → CodeBuddy 通过 Skill 生成日报（MD + PDF）
  → Skill 同时直接生成 4 个结构化 JSON 文件
  → upload_to_cloud.py 自动上传到微信云数据库（含回读校验）
  → 大老板打开小程序 → 看到真实数据 🦆
```

**你只需要做一次配置（大约30分钟），之后每天自动同步。**

### 关键数据流转路径

```
Skill 产出的 JSON 位置：
  workflows/investment_agent_data/miniapp_sync/
    ├── briefing.json    （简报页数据）
    ├── markets.json     （市场页数据）
    ├── watchlist.json   （标的页数据）
    └── radar.json       （雷达页数据）

上传脚本位置：
  .codebuddy/skills/touyanduck-daily/scripts/upload_to_cloud.py
```

## 二、Step 1：开通微信云开发

### 什么是"云开发"？

简单说，就是微信官方提供的一个"免费数据库"。我们把日报数据存到这个数据库里，小程序就能读到真实数据了。

### 操作步骤

1. **打开微信开发者工具**（就是你之前预览小程序用的那个工具）

2. **打开投研鸭项目**（`touyanduck_appid` 文件夹）

3. **点击工具栏上的「云开发」按钮**
   - 位置：顶部工具栏，一般在"编译"和"真机调试"旁边
   - 图标是一朵小云

4. **在弹出的窗口中点击「开通」**
   - 选择「按量计费」（有免费额度，你的用量远远用不完）
   - 环境名称填：`touyanduck`（或者随便起个名字）
   - 点击确认

5. **等待环境创建完成**（约1-2分钟）

6. **记下你的「环境ID」**
   - 创建完成后，在云开发控制台右上角会显示环境ID
   - 格式类似：`touyanduck-xxxxx` 或 `cloud1-xxxxxxx`
   - **把这个ID记下来**，后面要用！

> 免费额度有多少？
> 基础版每月100万次数据库读写。你的日报每天只写4条、大老板看几次，
> 连免费额度的0.1%都用不到，完全不用担心费用。

## 三、Step 2：创建数据库集合

### 什么是"集合"？

你可以把它想象成Excel里的4张表。每张表存一类数据。

### 操作步骤

1. 在云开发控制台，点击左侧「数据库」

2. 点击「添加集合」按钮，依次创建以下4个集合：

   | 集合名称 | 对应小程序页面 | 存什么数据 |
   |---------|-------------|-----------|
   | `briefing` | 简报页 | 核心事件、判断、行动建议、情绪分数 |
   | `markets` | 市场页 | 美股、M7、亚太、大宗、加密、GICS板块 |
   | `watchlist` | 标的页 | 各板块个股数据 |
   | `radar` | 雷达页 | 红绿灯、风险评分、聪明钱动向 |

3. **设置权限**：点击每个集合右侧的「权限设置」
   - 选择「所有用户可读，仅管理端可写」
   - 这样大老板能读数据，但不能改数据（只有你的脚本能写）

> 集合名称必须完全匹配（全小写，没有空格），否则数据读不到！

## 四、Step 3：获取密钥并配置

### 获取 AppSecret

1. 打开浏览器，访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 用你的微信扫码登录（管理员账号）
3. 左侧菜单找到：**开发管理** → **开发设置**
4. 找到「AppSecret(小程序密钥)」，点击「生成」
5. **用微信扫码确认**，然后复制 AppSecret
6. **妥善保存这个密钥！** 关闭窗口后就看不到了

> AppSecret 非常重要，相当于你的银行卡密码。千万不要分享给别人！

### 配置环境变量

打开终端（Mac上按 `Cmd + Space`，输入"终端"，回车），执行以下命令：

```bash
# 把下面的 xxx 替换成你的真实值
echo 'export WX_APPID="你的AppID"' >> ~/.zshrc
echo 'export WX_APPSECRET="你的AppSecret"' >> ~/.zshrc
echo 'export WX_CLOUD_ENV="你的云环境ID"' >> ~/.zshrc

# 让配置生效
source ~/.zshrc
```

### 验证配置

```bash
# 检查是否设置成功
echo $WX_APPID
echo $WX_APPSECRET
echo $WX_CLOUD_ENV
```

如果三行都有输出值，说明配置成功。

## 五、Step 4：修改小程序配置

打开 `touyanduck_appid/app.js`，找到 `globalData` 中的这两个字段：

```javascript
useCloud: false,           // 改成 true
cloudEnvId: '',            // 填入你的云环境ID
```

改成：

```javascript
useCloud: true,
cloudEnvId: '你的云环境ID',  // 例如 'touyanduck-xxxxx'
```

**就这一个地方要改，其他代码都不用动！**

## 七、日常使用

配置完成后，**你什么都不用做**！

每次你对 CodeBuddy 说"投资Agent"或"投资App"时，Skill 会自动：

1. 照常生成 MD + PDF 日报（完全不变）
2. 照常产出大老板消息文本（完全不变）
3. **直接生成 4 个结构化 JSON 文件**（AI 原生产出，非提取）
4. **自动上传到云数据库**（含回读校验，确保数据完整）
5. 大老板打开小程序就能看到最新数据

如果上传失败（比如网络问题），日报照常交付，小程序会用上一次的缓存数据。

## 八、架构说明

### 数据服务层（api.js v7.0）

小程序的数据服务层只接受两种数据来源：

| 优先级 | 数据源 | 说明 |
|--------|--------|------|
| 1（最高） | 微信云数据库 | 每天由 upload_to_cloud.py 上传的真实数据 |
| 2（降级） | 本地缓存 | 上一次成功拉取后保存的数据（30分钟有效期） |

**没有 Mock 数据降级**。如果云数据和缓存都不可用，页面会显示空状态提示，不会展示虚假数据。

### 文件结构

```
codebuddy-invest/
├── .codebuddy/skills/touyanduck-daily/
│   ├── SKILL.md                        # Skill 主控文档
│   ├── references/
│   │   ├── json-schema.md              # JSON Schema 规范（v5.9+）
│   │   └── data-collection-sop.md      # 数据采集 SOP
│   └── scripts/
│       ├── upload_to_cloud.py          # 上传脚本（v1.1，含回读校验）
│       └── requirements.txt            # Python 依赖
├── workflows/investment_agent_data/
│   └── miniapp_sync/                   # AI 每日产出的 4 个 JSON
│       ├── briefing.json
│       ├── markets.json

│       └── radar.json
└── touyanduck_appid/                   # 小程序源码
    ├── app.js                          # 入口（云开发配置在此）

    └── pages/                          # 4 个页面
        ├── briefing/
        ├── markets/
        ├── watchlist/
        └── radar/
```

## 九、常见问题

### Q: 上传失败怎么办？

不影响日报产出。上传是"旁路"操作，失败了小程序会继续用上一次的缓存数据。你可以之后手动重新上传。

### Q: 大老板打开小程序看到的是旧数据？

可能是你今天还没跑日报。小程序会优先显示当天数据，如果没有就显示最近一天的。

### Q: 要花钱吗？

微信云开发基础版完全免费，每月100万次读写。你的场景连1%都用不到。

### Q: 云不可用时小程序会显示什么？

会先尝试读取本地缓存（30分钟有效期）。如果缓存也过期或不存在，页面会显示空状态提示，**不会展示虚假数据**。

### Q: 数据安全吗？

数据在微信生态内流转（你的电脑 → 微信服务器 → 小程序），不会暴露到公网。AppSecret 存在你本地环境变量里，不会上传到代码仓库。

### Q: 云环境ID在哪里看？

微信开发者工具 → 点击「云开发」按钮 → 右上角显示的就是环境ID。

### Q: upload_to_cloud.py 的回读校验失败了？

v1.1 版上传脚本新增了回读校验功能。如果校验失败，脚本会输出具体哪些字段缺失或异常。常见原因：
- JSON 文件不完整（AI 生成过程中被中断）
- 网络波动导致部分数据上传失败

解决方法：重新运行上传命令即可。
