## 用户需求

投研鸭微信小程序阶段一前端体验升级，借鉴 Iran Briefing 的设计模式，在不改动后端的前提下，通过纯前端改造大幅提升信息密度和阅读体验。

## 产品概览

在现有4页面（简报/市场/标的/雷达）架构基础上，为简报页和雷达页注入3类全新信息模块，同步升级数据底栏，并将 JSON Schema 升级至 v1.3 以支撑新字段。

## 核心特性

1. **简报页 - 时间状态栏**：在顶部 hero 区域下方新增多时区（北京/美东）+ 开市状态 + 数据新鲜度的三栏状态条，借鉴 Iran Briefing 截图的时间状态栏设计
2. **简报页 - KEY DELTA 增量信息模块**：在核心事件卡片之前插入"增量信息 KEY DELTA"卡片，每条含状态标签（升级/新增/活跃/降温/稳定）+ 热度指示（1-5点）+ 一句话概述，让老板一眼看到"今天比昨天变了什么"
3. **简报页 - 核心判断扩展**：三大核心判断卡片新增关键决策者、参考源列表、概率标签、趋势方向等可选字段，对标 Iran Briefing 的 THINGS TO WATCH 深度卡片
4. **雷达页 - Fear and Greed 情绪卡片**：在红绿灯区域之后新增 CNN Fear and Greed 指数可视化卡片，含情绪条、当前值、昨日值、一周前值的趋势对比
5. **雷达页 - 预测市场预览卡片**：新增预测市场模块，展示 2-4 条预测概率条目（如"美联储6月降息 72%"），含来源标签和 24 小时趋势变化
6. **全局 - 数据底栏升级**：将简报页和雷达页底部的单行 dataTime 文本升级为双层状态栏，第一层显示数据源（云数据/本地缓存）+ 相对时间，第二层显示 source_type 标签（AI深度分析/实时行情）+ Skill 版本号
7. **Skill 规范 - JSON Schema v1.3**：在 json-schema.md 中为 briefing.json 新增 timeStatus / keyDeltas / coreJudgments 扩展字段，为 radar.json 新增 fearGreed / predictions 字段，为所有 JSON 新增 _meta 对象，所有新字段均为可选以保持向后兼容

## 技术栈

- 微信小程序原生开发（WXML + WXSS + JS）
- 现有组件体系：section-card / traffic-light / mini-chart / skeleton / stock-card
- 数据服务层：api.js（云数据库优先 + 本地缓存回退）
- 工具函数：format.js / color.js / animate.js
- Skill 规范体系：json-schema.md / data-collection-sop.md / SKILL.md

## 实现方案

### 整体策略

采用"Schema 先行 + 前端渐进增强"策略：先升级 json-schema.md 定义新字段契约，再逐页面实施前端改造。所有新字段设计为可选（前端用 `wx:if` 判空），确保旧数据照常渲染，零破坏性。

### 关键技术决策

1. **时间状态栏的时区计算**：在 `format.js` 中新增 `getMultiTimezone()` 函数，基于本地时间计算北京时间（UTC+8）和美东时间（UTC-5/UTC-4 夏令时），以及开市状态判断（美股 9:30-16:00 ET）。选择前端计算而非依赖后端传入，原因是时间是实时的而日报是静态的，前端计算能保证用户打开时看到的总是当前时刻的状态。

2. **KEY DELTA 热度点的渲染**：使用 `wx:for` 遍历固定 5 个点位，通过 `index < item.heat` 条件判断是否点亮，避免引入额外状态管理。这比用 CSS background-image 或 SVG 更轻量，符合小程序性能约束。

3. **coreJudgments 扩展字段的向后兼容**：新增的 keyActor / references / probability / trend 字段全部用 `wx:if` 包裹，不存在时不渲染对应 DOM 节点。`_applyData` 中对这些字段做默认值处理（空字符串/空数组），确保旧数据不报错。

4. **Fear and Greed 情绪条**：复用简报页已有的 `sentiment-bar` 设计模式（渐变轨道 + 指针点），但配色方案独立（绿色恐惧端 → 红色贪婪端），与简报页情绪仪表盘区分。使用 `animate.animateInteger` 实现数字跳动动画，保持体验一致性。

5. **预测市场概率条**：每条预测用水平进度条展示概率（0-100%），颜色根据概率值动态计算（低概率蓝色 → 高概率橙红色），24小时变化用箭头 + 数值展示趋势。

6. **数据底栏的 source_type 判断**：从 JSON 数据的 `_meta.sourceType` 字段读取，若不存在则降级为不显示该行，保持旧数据兼容。相对时间（如"3分钟前"）通过 `format.js` 新增的 `getRelativeTime(dateString)` 函数计算。

## 实现注意事项

- **动画一致性**：新增模块统一使用项目已有的 `stagger-item` + `stagger-show` 动画模式，简报页用 `slideUp`，雷达页用 `slideInRight`，保持页面动画风格统一。animation-delay 按照插入位置在现有链条中顺延，避免与已有模块冲突。
- **setData 批量调用**：新增字段全部在已有的 `_applyData` 函数单次 `setData` 调用中一并设置，避免多次 setData 导致的渲染抖动。
- **样式隔离**：新增 CSS 类名全部使用语义化前缀（`ts-` 时间状态栏 / `kd-` KEY DELTA / `fg-` Fear Greed / `pm-` 预测市场 / `ds-` 数据状态栏），避免与现有类名冲突。
- **向后兼容验证**：所有新增前端渲染块均包裹在 `wx:if` 中，当对应数据字段不存在或为空时完全不渲染，确保现有云数据库中的旧数据能正常展示。

## 架构设计

### 数据流（新增字段）

```

    ↓

    ↓
api.js fetchData() → 页面 _applyData()
    ↓                    ↓
新字段存在 → 渲染新模块   新字段不存在 → wx:if 跳过
```

## 设计风格

延续投研鸭小程序现有的专业金融终端风格，在深蓝黑(#1a1a2e)与金色(#FFB800)的主色调基础上，借鉴 Iran Briefing 的信息密度设计理念，通过结构化卡片和视觉层级引导用户快速获取关键增量信息。

### 时间状态栏（简报页 hero 下方）

紧贴 hero 区域底部，无间距融合。采用三等分布局，深色半透明背景(rgba(26,26,46,0.85))，与 hero 渐变色形成视觉连贯。左栏北京时间（大字时间+小字"Beijing"标签），中栏美东时间（大字时间+开市状态标签，绿色表示交易中/灰色表示已收盘），右栏数据更新（相对时间+刷新频率标签）。整体高度约 120rpx，字号分层：时间数字 32rpx 加粗，标签文字 20rpx 辅助灰色。

### KEY DELTA 增量信息模块（简报页核心事件前）

使用 section-card 组件包裹，标题"增量信息 KEY DELTA"。每条 delta 项为横向布局：左侧状态标签（胶囊形，不同状态对应不同底色：升级-橙红、新增-蓝色、活跃-金色、降温-灰色、稳定-绿色），中间标题文本（28rpx加粗），右侧热度指示（5个圆点排列+文字标签"加速中/活跃/降温"）。点亮的圆点为橙红色(#FF6B35)，未点亮为浅灰色(#e0e0e0)。每条之间用 1rpx 分割线隔开。底部显示更新时间小字。

### 核心判断扩展行（简报页三大判断卡片内）

在现有"置信度条 + 逻辑文本"之后追加扩展信息区域。当存在关键决策者时，显示"关键决策"图标+决策者姓名行。当存在概率和趋势标签时，以胶囊标签组形式水平排列（"高可能性"蓝底白字、"上升"绿底白字带箭头、"高影响"红底白字）。当存在参考源时，显示"REFERENCE"标签+来源列表+数量角标，样式紧凑。扩展区域与主内容区之间有 1rpx 虚线分隔，视觉上属于同一卡片但信息层级更低。

### Fear and Greed 情绪卡片（雷达页红绿灯后）

使用 section-card 包裹，标题"市场情绪指数"。卡片内部分三区：顶部情绪条（渐变轨道从绿色恐惧到红色贪婪，带指针点标记当前值位置），中部大字显示当前分数（48rpx加粗）+情绪标签（如"Fear"圆角背景），底部三栏对比（昨日/一周前/一月前值），变化方向用小箭头和颜色标识。整体设计复用简报页 sentiment-bar 的视觉语言但独立配色。

### 预测市场预览卡片（雷达页 Fear and Greed 后）

使用 section-card 包裹，标题"预测市场"，可折叠。每条预测项为卡片式布局：标题行（预测问题文本+来源标签如"Polymarket"小字），概率行（水平进度条+百分比数字），变化行（24小时变化值+趋势箭头）。进度条颜色根据概率值渐变（低概率淡蓝→高概率橙色）。条目之间 16rpx 间距。

### 数据状态底栏（简报页+雷达页底部）

替换现有单行 footer，升级为双层结构。第一层：左侧状态圆点（云数据绿色/缓存橙色）+数据源文字+相对时间文字，居中对齐。第二层：source_type 标签（"AI深度分析"金色底/"实时行情"蓝色底）+Skill版本号，居中对齐。两层之间 4rpx 间距，整体字号 20rpx，颜色 #bbb 辅助灰。
