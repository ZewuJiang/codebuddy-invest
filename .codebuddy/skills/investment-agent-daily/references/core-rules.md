## 成稿终审铁律（3条 — v17.8新增 — RULE FIVE）

> **背景**：v17.7版本出现过PLTR行数据写成"收涨/+正"的模糊描述而非精确数值的低级错误，直接被大老板发现。此类错误严重损害报告公信力，绝不允许再次发生。

| # | 规则 | 说明 | 违反后果 |
|---|------|------|---------|
| **20** | **逐单元格数据扫描（v17.8）** | 报告成稿后，**逐行逐单元格**检查所有表格：每个"收盘价"列必须是`$XXX.XX`格式精确数值，每个"涨跌幅"列必须是`+X.XX%`或`-X.XX%`格式。发现任何非数值文字（如"收涨""上涨""正""跌""约""小幅"等）→立即搜索补全精确数据后才允许输出 | **致命——报告不得交付** |
| **21** | **数据缺失宁迟勿糊（v17.8）** | 采集阶段如某标的精确数据暂时无法获取，**必须至少尝试3个以上数据源**（Google Finance → Yahoo Finance → 新浪财经 → 东方财富 → StockAnalysis → web_search多关键词），穷尽搜索后仍无法获得精确值→**删除该行**（宁可少一行也不用模糊文字凑数） | **致命——报告返工** |
| **22** | **三轮终审复核（v17.8）** | 撰写完成后强制执行三轮复核：①**数据轮**：逐表格逐单元格确认精确数值（无模糊、无空位、无"~约"）；②**逻辑轮**：§1核心结论与§2-§5数据一致性交叉比对，涨跌方向/幅度无矛盾；③**格式轮**：禁用词扫描+行数检查+markdown语法正确。三轮全通过后一次性输出最终版 | **致命——报告不得交付** |

## 自主进化规则（2条 — v18.0新增 — RULE SIX）

> **背景**：v17.0到v17.9的所有进化都依赖"被动踩坑→人工反馈→手动修复"模式。v18.0引入自主反思引擎，实现"主动反思→自动提案→分级执行→审计回滚"的自主进化闭环。核心原则：**质量只升不降，宁可不进化也不退化。**

| # | 规则 | 说明 | 违反后果 |
|---|------|------|---------|
| **29** | **反思引擎强制执行（v18.0）** | 每次日报第四阶段交付完成后，**必须**进入第五阶段自主反思引擎。按六维框架（数据源效率/新闻源发现/分析框架/覆盖盲区/流程效率/格式呈现）逐一快速检查。无论是否有进化提案，都必须在聊天窗口输出反思总结+在evolution-log.md记录。**禁止跳过第五阶段** | **致命——进化闭环断裂** |
| **30** | **质量铁栅栏（v18.0）** | 任何进化提案必须通过三项强制检验：①准确性不降级 ②完整性不降级 ③实时性不降级。三级分级执行：🟢低风险自动执行+通知、🟡中风险默认生效（日报末尾报告，用户不反对则下次自动固化）、🔴高风险需用户明确确认。所有变更必须记录精确Diff以支持回滚。分级判断模糊时永远向上升级一级（保守原则）。详见[evolution-rules.md](evolution-rules.md) | **致命——高风险未确认执行/绕过铁栅栏/无Diff记录** |

## 数据禁用表述对照（v17.8升级）

| 禁用表述 | 替换为 | 示例 |
|----------|--------|------|
| "—" / "-" / "N/A" | 精确数值 | "—" → "$182.37" |
| "~$264" / "约$264" | "$264.72" | 去掉波浪号和"约" |
| "小幅上涨" | "+0.20%" | 精确涨跌% |
| "略有下跌" | "-1.68%" | 精确涨跌% |
| "VIX回落至22-23" | "VIX收21.44 (+7.96%)" | 精确值+精确涨跌% |
| "MSFT ~$407" | "MSFT $398.55 (+1.48%)" | Google Finance精确值 |
| **"收涨" / "收跌"** | **"$153.50"** | **v17.8新增：必须是精确收盘价** |
| **"+正" / "-负" / "上涨" / "下跌"** | **"+1.25%" / "-2.55%"** | **v17.8新增：必须是精确涨跌幅** |
| **"涨超X%" / "跌超X%"** | **"+9.26%" / "-3.14%"** | **v17.8新增：在表格单元格中必须精确** |
| **任何中文描述性文字出现在数值列** | **精确数值** | **v17.8新增：收盘价列只允许$XXX.XX，涨跌幅列只允许±X.XX%** |
