# 报告格式指南（v18.0）

> **用途**：投资Agent每日策略简报的报告撰写格式规范、覆盖范围、PDF渲染要求、进化提案格式。
> **核心原则**：GS/MS/MBB投行简报风格，信息密度优先，200-300行。

---

## 一、§1 今日核心结论

**目的**: 结构化呈现过去24h最重要的事+行动建议

**呈现风格**: GS/MS/MBB投行简报风格，浅蓝底色卡片，左对齐，层次清晰。结论行字体加大突出，放在所有信息之后。

**⚠️ 灵活原则（最高优先级）**：模块不是死板模板，一切以当日实际市场动态为准。有什么写什么，没什么就不写。市场平淡时可能只需2-3个模块+结论。

**可选模块菜单**（根据当日实际情况灵活选用，非必填）：
- **事件链**：重大事件因果传导
- **全球资产反应**：核心资产价格变动
- **市场分化**：不同市场走势对比
- **叠加因素**：其他影响因素
- **正向/负向锚**：关键利好或利空
- **📌 结论**：最终判断+操作建议（**必写**，永远放最后）

**格式示例**:

```markdown
## §1 今日核心结论

> **🚨 [核心事件标题]**
>
> **事件链**：
> - [因果链条]
>
> **全球资产反应**：
> - **[资产1]**：[数据+涨跌]
> - **[资产2]**：[数据+涨跌]
>
> **📌 结论**：[最终判断，**加粗关键操作建议**]

### 三大核心判断

| # | 判断 | 置信度 | 逻辑链 |
|---|------|--------|--------|
| 1 | **[最重要判断]** | XX% | [因果链] |
| 2 | **[次重要判断]** | XX% | [因果链] |
| 3 | **[第三判断]** | XX% | [因果链] |

### 行动建议

| 时间维度 | 建议 | 理由 |
|----------|------|------|
| 今日 | [具体可执行建议或"无需操作"] | [简洁理由] |
| 本周 | [关注要点] | [简洁理由] |
```

**规则**:
- 行动建议必须可执行（"买入XX""减仓至XX%""无需操作"），禁止"关注""注意"
- 无重大事件时可直接写"市场平稳运行，无需操作"

## 二、§2 全球市场隔夜速览

**目的**: 分区域表格快速了解全球市场

**格式**:

```markdown
## §2 全球市场隔夜速览

### 美股（收盘 MM/DD）

| 指数/标的 | 收盘价 | 涨跌幅 | 关键信号 |
|-----------|--------|--------|---------|

> **美股信号**: [一段话总结]

### 亚太（收盘/盘前 MM/DD）

| 指数 | 最新价 | 涨跌幅 | 关键信号 |
|------|--------|--------|---------|

> **亚太信号**: [一段话总结]

### 大宗 & 汇率 & 加密

| 品种 | 最新价 | 涨跌幅 | 关键信号 |
|------|--------|--------|---------|

> **大宗信号**: [一段话总结]

### 宏观一句话

> **[当前宏观环境最核心判断，≤30字]**
```

**规则**: 已收盘用精确收盘数据，未收盘标注"盘前"。涨跌幅公式计算。"关键信号"列≤15字。禁止flag emoji。

**⚠️ 原油指标规范（v18.3）**：原油数据**统一以布伦特（Brent）为主指标**，WTI为辅。具体要求：
- §1核心结论：标题和事件链以布伦特价格+涨跌幅为叙事锚，WTI可括号补充
- §2大宗表格：布伦特排在WTI前面
- §5红绿灯和监控阈值：原油阈值以布伦特标尺设定
- 全文叙事中出现"油价"时，优先引用布伦特数据
- **原因**：布伦特是全球原油定价基准（覆盖全球2/3以上原油交易），WTI仅为美国本土基准

## 三、§3 重点标的与行业分析

**核心原则**: 信息密度优先。核心个股表格+GICS板块每期必出，重点标的展开有事才写。

**三层结构**:
1. **核心个股表格**（每期必出）：M7 + 重点AI标的 + 当日焦点个股
2. **GICS 11板块表现**（每期必出）：全部11板块排名 + 轮动解读
3. **重点标的展开**（有事才写）：四栏卡片（事件/关键数据/为什么重要/投资含义）

**AI全产业链24环节覆盖**: 详见 `ai-supply-chain-universe.md`

**其他覆盖范围**:

### 纳入标准（满足任一即报告）

- 重大订单（$10B+）或战略合作
- 财报超预期或大幅miss
- 管理层重大变动
- 产能/技术突破

- 供应链重大中断

- 未上市AI公司重磅产品发布/大额融资/IPO申请

**⚠️ RULE TWO**: §3只展开个股/公司事件，不展开机构/基金动向（那属于§4）

## 四、§4 基金&大资金动向

**目的**: 追踪全球头部基金和投资大师的最新动向及核心观点。详见 `fund-universe.md`。

**三梯队覆盖**:
- **一级核心（8家）**: Berkshire/BlackRock/Vanguard/Bridgewater/Blackstone/ARK/段永平H&H/高瓴 — 有动态必报
- **二级追踪（10家）**: Millennium/Citadel/Point72等 — 季度13F+重大发言必报
- **三级雷达（8家）**: Marshall Wace/Balyasny等 — 有重大事件才报

**格式（有动态时）**:

```markdown
## §4 基金&大资金动向

| 机构/人物 | 动态 | 时间 | 投资含义 |
|-----------|------|------|---------|

> **动向解读**: [一段话总结]
```

**格式（无动态时）**: 写"过去24小时无重大资金动向"即可。

**⚠️ RULE TWO**: §4只写机构/大师动向，不重复§3已展开的公司基本面事件

**特殊检测**: 北向资金持续大额净买入+指数不涨 → 必须标注"内资派发嫌疑"

## 七、PDF渲染铁律

- CSS `font-family`: `"STHeiti", "Hiragino Sans GB", "Noto Sans CJK SC", "PingFang SC", sans-serif`
- 必须使用`md_to_pdf.py`两轮渲染法（Probe测高度→精确高度渲染）
- 严禁flag emoji（如🇨🇳🇺🇸），用文字替代
- 表格单元格≤25字，最多7列
- 页面宽度280mm
- 严禁pandoc直接转PDF / PingFang SC排首位 / pypdf裁剪mediabox

---

## 八、撰写原则

- §1核心结论：结构化bullet point，结论放最后
- §2市场速览：只报已收盘精确数据，未开盘标注"盘前"。每个单元格必须有精确数值
- §3重点标的：核心个股表格+GICS板块每期必出，重点标的展开有事才写
- §4聪明钱：有动向才详写，无动向一句话带过
- §5风险雷达：红绿灯+日历+风险提示，精简有力
- **全文200-300行，阅读时间约5分钟**

## 九、大老板消息文本（第四阶段交付产物）

**目的**: 每次生成策略简报后，在聊天窗口额外输出一段可直接复制发送给大老板的纯文本消息。

**来源**: 取自 §1 今日核心结论的引用块（`>` 部分）。

**固定格式**:

```
老板好～今日二级策略简报结论如下：

{§1核心结论标题，去掉emoji和markdown语法}

事件链：

{事件链内容，"-"列表保留}

{模块1标题}：

{模块1内容，"-"列表保留}

{模块2标题}：

{模块2内容}

...更多模块按§1实际内容灵活增减...

📌 结论：{结论内容，去掉markdown加粗语法，保留纯文本}
```

**格式规则**:

| 规则 | 说明 |
|------|------|
| 来源 | 取自 §1 今日核心结论的引用块（`>` 部分） |
| 去掉markdown语法 | `**加粗**` → 纯文本，`>` 引用符号去掉，`🚨` 等emoji去掉 |
| 保留结构 | 保留 `-` 列表符号、模块标题冒号、📌 结论emoji |
| 固定开头 | `老板好～今日二级策略简报结论如下：` 每天一样 |
| 不含三大判断表格 | 只取引用块部分，不含"三大核心判断"和"行动建议"表格 |
| 纯文本 | 可直接粘贴到微信/消息发送，无任何markdown格式 |
| 输出位置 | 聊天窗口中独立代码块展示，方便直接复制 |

## 十、进化提案模块（第五阶段交付产物 — v18.0新增）

**目的**: 每次日报生成后，在大老板消息文本**之后**，输出第五阶段反思引擎的结果，包括反思总结和进化提案（如有）。

**位置**: 大老板消息文本代码块之后，作为独立区块。

**注意**: 此模块仅在聊天窗口中呈现，**不写入MD/PDF报告文件**（日报PDF保持简洁，不含进化内容）。

### 10.5 格式规则汇总

| 规则 | 说明 |
|------|------|
| 输出位置 | 聊天窗口中，大老板消息文本代码块**之后** |
| 不写入报告 | 此模块不写入MD/PDF报告文件（报告保持简洁） |
| 每次必输出 | 无论是否有提案，都必须输出反思总结（"无提案"也是有效输出） |
| 分隔线 | 用`---`分隔大老板消息文本和反思引擎输出 |
| 累计统计 | 每次输出底部附带累计反思次数和已固化提案数 |
| 多条提案 | 如有多条提案，按级别从高到低排列（🔴→🟡→🟢） |
