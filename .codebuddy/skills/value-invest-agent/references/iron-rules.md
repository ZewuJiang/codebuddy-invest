# 核心原则（40条铁律）

> ### 🚨🚨🚨 RULE ZERO — 全局数据查证铁律（v9.1 升级，最高优先级）
>
> **禁止 AI 凭训练数据中的模糊记忆直接输出任何数字。**本报告是呈现大老板的深度研究报告。报告中的每一个数据、每一个事实陈述、每一个市场信息，都必须且只能来源于当期实时搜索结果（web_search / web_fetch）。所有数值类数据（价格/涨跌幅/汇率/市值/PE/营收/利润/增速/持仓权重/目标价等）必须从权威数据源实时查证，绝不凭记忆输出。自查三问：①来自本次哪次搜索？②搜索结果原文怎么写的？③时间戳是否合理？——任一答不上来→凭记忆编造→必须重新搜索。**宁缺毋错，宁空不编。**训练数据仅允许用于：确定搜索方向/理解搜索结果背景/分析方法论框架。
>
> **典型事故（2026-04-06）**：离岸人民币 AI 记忆 7.28 实际 6.88 差 5.8%；持仓权重 AAPL 记忆 28% 实际 22.6%——凡凭记忆输出的数字，一旦被发现偏差，等同数据造假。

1. **数据一手性**: 财报数据优先从公司官方IR（投资者关系）网站获取，确保一手准确
2. **内容严谨性**: 所有数据、数字、百分比必须有出处，严禁凭记忆臆造
3. **⚠️⚠️ 数据实时性（最高优先级）**: **所有市场数据（市值、股价、排名、竞对市值等）必须通过实时搜索获取最新数据，严禁使用AI记忆中的历史数据或混用不同时间点的数据**。具体要求：
   - a) 市值排名：必须从 companiesmarketcap.com 或同等实时数据源获取**同一时间点**的排名数据
   - b) 竞对对比表中所有公司的市值、股价等数据必须来自**同一数据源、同一时间截面**
   - c) 报告中出现的市值 vs 排名必须逻辑自洽
   - d) **绝对禁止**：用数据源A的公司X数据 + 数据源B的公司Y数据拼凑排名
   - e) 每个数据点必须标注数据来源和时间
   - f) **写入报告后立即做逻辑自洽验证**
4. **分析全面性**: 不遗漏任何关键维度，从定量到定性全方位覆盖
5. **观点清晰性**: 每个分析模块都要有明确的结论和判断，不模棱两可
6. **论据充分性**: 每个观点都要有充分的数据和事实支撑
7. **多源交叉验证**: 关键数据至少2个独立数据源确认
8. **长周期视角**: 至少复盘过去5年或10年的业务历程与财务表现
9. **财报深度拆解**: 对最近4-8个季度财报逐季分析，年报逐年对比
10. **⚠️ 数学验证**: 所有计算必须逐步展示并验算
11. **⚠️ 全文一致性**: 同一数据在报告不同位置引用时必须严格一致
12. **⚠️ 数据逻辑自洽**: 数据之间必须逻辑一致——数值与排名不矛盾
13. **⚠️⚠️ 市场预期合理性验证（v4）**: 引用"市场一致预期"时，必须通过至少2个数据源交叉验证。验证方法：预期值 ≈ 上年同期 ×(1 + 分析师一致增速预期)。偏差>5%需标注说明
14. **⚠️ 近似数据统一表述（v4）**: 同一指标在全文不同位置使用的近似值必须统一
15. **⚠️ 止损位全文一致性（v4）**: 止损价位在所有出现位置必须完全统一
16. **⚠️ 分析师目标价表述规范（v4）**: 严格区分"一致预期均值"、"目标价区间"、"个别分析师目标价"
17. **⚠️⚠️ 货币单位全文一致性（v6）**: 全文金额单位统一使用目标公司本币；跨市场竞对对比时折算为同一货币并标注汇率
18. **⚠️ 会计准则标注（v6）**: 报告开头必须标注会计准则，涉及准则差异显著指标时需特别说明
19. **⚠️⚠️ 一手财报优先级（v6）**: ① 公司官方IR → ② 对应证监会/监管机构 → ③ 三方平台
20. **⚠️⚠️ 图表与文本数据同步（v7）**: 文本表格数据与chart_data JSON必须使用同一季度数据
21. **⚠️ 计算过程完整展示（v7）**: 所有关键计算结果必须附完整推导链
22. **⚠️ 同比增速使用官方口径（v7）**: 优先使用公司官方口径，自行计算须附完整公式
23. **⚠️ 三方数据差异标注（v7）**: 手动计算值与第三方数据存在差异时必须标注差异来源
24. **⚠️⚠️ PE历史分位数必备（v7）**: Section 1.3必须包含PE(TTM)在历史区间的百分位分析
25. **⚠️ 盘前/盘后数据标注（v7）**: 盘前/盘后数据必须标注时间并提示波动大
26. **⚠️⚠️ 分析师预期增速跳跃必须解读（v7）**: 相邻年度增速显著变化(>10pp)必须附专门解读
27. **⚠️⚠️⚠️ 股价区间涨跌幅必须从实时数据源直接获取（v7.1，最高优先级）**: 1M/6M/1Y/YTD涨跌幅**必须从Finviz.com直接读取Perf Month/Perf Half Y/Perf Year字段，严禁自行推算**。同时获取SPY和QQQ对比基准。
28. **⚠️⚠️ 毛利润增速 vs 营收增速校验（v8）**: So What必须计算并比较两者增速
29. **⚠️⚠️ FCF质量拆解（v8）**: FCF正但EBITDA下降时必须拆解FCF构成来源
30. **⚠️⚠️ Executive Summary必须包含SBC稀释保守中枢（v8）**: 综合评级必须给出双中枢
31. **⚠️⚠️ 报告末尾必须有"核心矛盾一句话提示"框（v8）**: 免责声明前必须有
32. **⚠️⚠️ Bull Case每条必须附论据（v8）**: 每条必须附1-2句具体论据支撑
33. **⚠️⚠️ Section 8跟踪指标表必须包含毛利率维度（v8）**: 乐观+悲观两个触发条件
34. **⚠️⚠️ So What段的数据量化充分性（v8）**: 每个核心观点至少1个具体数字量化
35. **⚠️⚠️⚠️ Non-IFRS/Non-GAAP调整项逐项拆解（v9，高优先级）**: 必须从年报提取逐项调整差异表（SBC/无形摊销/减值/投资损益/重组费用），每项必须精确金额。DCF模型参数须与此表一致
36. **⚠️⚠️ D&A精确拆解必须从年报附注获取（v9）**: 从年报附注提取三大分项（物业设备折旧+使用权折旧+无形摊销），**严禁从EBITDA反推**
37. **⚠️⚠️ 多口径数据必须校准并标注口径（v9）**: SBC(Non-IFRS调整口径 vs 员工福利总口径)、CapEx(P&L口径 vs 现金流量表口径)等必须标注口径，全文统一
38. **⚠️⚠️ 历史财报PDF逐年获取（v9）**: 5-10年历史数据不允许仅依赖三方平台或估算，必须逐年从公司IR获取年报PDF提取精确值
39. **⚠️⚠️⚠️ 估计值标记与补全闭环（v9，最高优先级）**: 报告定稿前"~"/"(估)"标记必须经过至少3轮搜索尝试精确化。最终"(估)"标记≤总数据点的5%
40. **⚠️⚠️ SOTP估值法对多元化公司为必备（v9）**: 持有3+独立业务板块或大量投资组合的公司，SOTP为必备估值方法（含各板块独立估值+投资组合估值+HoldCo Discount）

## Non-IFRS/Non-GAAP调整项标准采集模板

> 对于IFRS/US GAAP公司，**必须**在Section 1.5或Section 4中包含以下调整表：

```markdown
| Non-IFRS调整项 | 金额(亿) | 说明 |
|---------------|----------|------|
| IFRS净利润 | {X} | 按IFRS准则 |
| (+) SBC（股份支付） | {A} | Non-IFRS调整口径 |
| (+) 无形资产摊销 | {B} | 并购产生的无形资产 |
| (+/-) 投资公司公允价值变动 | {C} | 处置收益/减值 |
| (+) 减值损失 | {D} | 商誉及其他减值 |
| (+/-) 其他Non-IFRS调整 | {E} | 税务调整等 |
| **Non-IFRS净利润** | **{Y}** | **管理层口径** |

> 验证：Y = X + A + B + C + D + E（误差≤1%）
```
