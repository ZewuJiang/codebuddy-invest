# 财报分析框架执行指南 (Analysis Framework Guide)

> **用途**: 提供标准化的财报分析执行流程  
> **适用对象**: 所有使用earnings-report-analysis Skill的分析任务  
> **执行标准**: 机构投资者级别分析深度

---

## 执行流程总览(4阶段,约3-4小时)

```
阶段1: 数据收集与验证 (90分钟)
  ├── 官方数据源定位 (15分钟)
  ├── 最新季度数据提取 (30分钟)
  ├── 历年营收数据收集 (30分钟)
  └── 三方交叉验证 (15分钟)

阶段2: 业务结构分析 (60分钟)
  ├── 分部营收拆解 (30分钟)
  ├── 地区/产品拆解 (20分钟)
  └── 关键驱动因素识别 (10分钟)

阶段3: 管理层信息提取 (45分钟)
  ├── 下季度指引解读 (15分钟)
  ├── 全年展望提取 (15分钟)
  └── 核心亮点总结 (15分钟)

阶段4: 风险与投资建议 (45分钟)
  ├── 风险矩阵构建 (20分钟)
  ├── 投资价值评估 (15分钟)
  └── 目标价计算 (10分钟)
```

### 1.1 官方数据源定位(15分钟)

#### 任务目标

### 1.2 最新季度数据提取(30分钟)

#### 任务目标
从官方文件提取最新季度的核心财务数据

#### 必须提取的数据(9项)

| # | 数据项 | 英文名称 | 提取位置 | 计算验证 |
|---|--------|---------|---------|---------|
| 1 | **总营收** | Revenue / Net Sales | Consolidated Statements of Operations,第1行 | - |
| 2 | **营业成本** | Cost of Revenue | 利润表 | - |
| 3 | **毛利润** | Gross Profit | 利润表 | = 总营收 - 营业成本 |
| 4 | **毛利率** | Gross Margin | 利润表(或需计算) | = 毛利润 / 总营收 × 100% |
| 5 | **营业利润** | Operating Income | 利润表 | - |
| 6 | **营业利益率** | Operating Margin | 利润表(或需计算) | = 营业利润 / 总营收 × 100% |
| 7 | **净利润** | Net Income | 利润表,底部 | - |
| 8 | **稀释EPS** | Diluted EPS | 利润表,最底部 | = 净利润 / 稀释股本 |
| 9 | **自由现金流** | Free Cash Flow | 现金流量表 | = 经营现金流 - 资本开支 |

#### 同比/环比增长率计算

```python
# 必须提取的对比数据
current_quarter = Q1_2026  # 当前季度
same_quarter_last_year = Q1_2025  # 去年同期(同比对比)
previous_quarter = Q4_2025  # 上一季度(环比对比)

# 同比增长率(YoY)
yoy_growth = (current_quarter - same_quarter_last_year) / same_quarter_last_year * 100

# 环比增长率(QoQ)
qoq_growth = (current_quarter - previous_quarter) / previous_quarter * 100
```

#### 输出模板

```markdown
## 一、核心财务数据

| 指标 | 26Q1 | 同比(YoY) | 环比(QoQ) |
|------|------|----------|----------|
| **总营收** | **$XX.X B** | **+XX.X%** | +XX.X% |
| **净利润** | **$XX.X B** | **+XX.X%** | +XX.X% |
| **Non-GAAP EPS** | **$X.XX** | **+XX.X%** | +XX.X% |
| **毛利率** | **XX.X%** | +X.Xppt | +X.Xppt |
| **营业利益率** | **XX.X%** | +X.Xppt | +X.Xppt |
| **自由现金流** | **$XX.X B** | **+XX.X%** | — |

> 💡 [核心洞察一句话总结,必须有具体数据支撑]

**数据来源**: [SEC Form 10-Q, 发布日期YYYY-MM-DD]
**对比基准**: 
- 同比: 2025年Q1 (营收$XX.XB)
- 环比: 2025年Q4 (营收$XX.XB)
```

**关键注意事项**:
- ⚠️ 如公司披露GAAP和Non-GAAP两套数据,**优先使用Non-GAAP**(剔除一次性项目)
- ⚠️ 环比对比时,注意季节性因素(如iPhone公司Q1通常低于Q4)
- ⚠️ 毛利率变动使用**百分点(ppt)**,而非百分比

#### 快速获取方法

**方法1: 从最新10-K的对比表获取**
```
定位: Form 10-K → Item 8 → Financial Statements → Consolidated Statements of Operations
通常包含: 最近3年的年度对比数据
```

**方法2: 从公司IR网站批量获取**
```
大多数公司IR网站有 "Financial History" 或 "Historical Financials" 页面
示例: 
- 台积电: https://investor.tsmc.com/english/quarterly-results/revenue-info
- 博通: https://investors.broadcom.com/financial-information/quarterly-results
```

**方法3: 从权威第三方平台获取(需验证)**
```
- Bullfincher: https://bullfincher.io/companies/[ticker]/revenue
- Stock Analysis: https://stockanalysis.com/stocks/[ticker]/revenue/
```

对于10年历年数据,必须通过以下验证:

## 阶段2: 业务结构分析(60分钟)

### 2.1 分部营收拆解(30分钟)

#### 任务目标
拆解公司营收结构,识别各业务板块的增长驱动力

#### 数据提取位置

**美股公司**(Form 10-K/10-Q):
```
定位: Item 1 → Segment Information
或: Item 8 → Note X - Segment Information (财务报表附注)
```

**通常披露内容**:
- 各业务板块营收
- 各业务板块营业利润
- 各业务板块资产

#### 必须分析的维度

| 维度 | 必须包含 | 深度分析要求 |
|------|---------|-------------|
| **各板块营收** | 金额+占比 | 同比/环比增长率 |
| **增长驱动** | 具体产品/客户 | 定量分析(如"AI芯片$8.4B,+106%") |
| **毛利率**(如有) | 各板块毛利率 | 与总体对比 |
| **结构变化** | 占比变化趋势 | 与去年同期对比 |

#### 输出模板

```markdown
## 四、主营业务收入(按业务板块)

| 板块 | 26Q1营收 | 营收占比 | 同比(YoY) | 环比(QoQ) | 核心驱动 |
|------|---------|---------|----------|----------|---------|
| ✅ **板块A** | **$XX.X B** | **XX%** | **+XX%** | +XX% | [具体产品名称]+[客户名称]+[数据] |
| **板块B** | **$XX.X B** | **XX%** | **+XX%** | +XX% | [具体产品名称]+[客户名称]+[数据] |
| **- 子板块B1** | **$XX.X B** | **XX%** | **+XXX%** | — | [具体说明] |
| **- 子板块B2** | **$XX.X B** | **XX%** | **+XX%** | — | [具体说明] |
| **板块C** | **$XX.X B** | **XX%** | **+XX%** | +XX% | [具体产品名称]+[客户名称]+[数据] |

**关键洞察**:
- **板块A营收($XX.XB)已是板块B的X.X倍**,成为绝对主导
- 板块增速对比:最快板块A(+XX%),最慢板块C(+XX%)
- **结构性变化**:板块A占比从XX%(25Q1)提升至XX%(26Q1),**同比+Xppt**
- [其他重要发现,必须有数据支撑]

**数据来源**: [SEC Form 10-Q, Item 1 - Segment Information]
```

**关键注意事项**:
- ✅ 使用 ✅ emoji标记最重要/增速最快的板块
- ✅ 子板块拆解使用缩进(如"- 子板块B1")
- ✅ 营收占比合计必须=100%
- ✅ 核心驱动必须具体到产品/客户名称

**反面案例**:
- ❌ "板块A增长强劲" ← 缺乏数据
- ✅ "板块A同比+52%,AI半导体营收$8.4B(+106%)占比53%"

### 2.3 关键驱动因素识别(10分钟)

#### 任务目标
从管理层讨论(MD&A)中提取业务增长的核心驱动因素

#### 提取位置
```
Form 10-Q/10-K → Item 2 (10-Q) / Item 7 (10-K) → Management's Discussion and Analysis (MD&A)
```

#### 必须回答的问题

1. **哪些产品/服务驱动了营收增长?**
   - 定量: 具体营收贡献(如"AI芯片$8.4B")
   - 定性: 为什么这些产品增长(技术优势/市场需求)

2. **哪些客户驱动了营收增长?**
   - 是否有新增大客户?
   - 现有客户订单变化?

3. **有哪些外部因素?**
   - 行业周期位置(上行/下行/底部)
   - 政策影响(如AI投资周期)
   - 宏观经济(利率、汇率)

4. **有哪些内部因素?**
   - 产能扩张(CapEx投资)
   - 并购整合
   - 技术突破

#### 输出格式

```markdown
### 核心驱动因素分析

**外部驱动**:
1. **AI基础设施投资周期**: 超大规模云客户(Google、Meta、Amazon等)26-27年CapEx高峰,单家年度投资$50-75B
2. **定制XPU需求爆发**: 超大规模云客户从通用GPU转向定制加速器,降低单位算力成本30-50%

**内部驱动**:
1. **供应链锁定至FY 2028**: 提前3年锁定先进制程晶圆、HBM、基板,建立竞争壁垒
2. **AI网络芯片技术领先**: Tomahawk 6交换芯片100 Tbps带宽,领先竞争对手1代

**客户驱动**:
1. **定制XPU客户从5家→6家**: Google TPU第七代已部署,Meta定制加速器近期交付
2. **单一客户营收占比**: Nvidia约占总营收20%,持续增长

**数据来源**: [SEC Form 10-Q, Item 2 - MD&A, Page XX-XX]
```

### 3.1 下季度指引解读(15分钟)

#### 任务目标
提取管理层对下一季度的财务指引(Guidance)

**方法1**: 财报新闻稿(Earnings Release)
```
通常在新闻稿的 "Business Outlook" 或 "Q2 2026 Guidance" 部分
```

**方法2**: 财报会议实录(Earnings Call Transcript)
```
CEO/CFO的Opening Remarks部分
或Q&A环节回答分析师提问
```

**方法3**: Form 8-K
```
Exhibit 99.1 - Press Release
通常包含完整指引
```

#### 必须提取的指引项

| 指引项 | 必须包含 | 备注 |
|--------|---------|------|
| **营收指引** | 区间或中点(如"$390-402亿") | 必须标注 |
| **环比增速** | 约+XX% | 相对上一季度 |
| **同比增速** | 约+XX% | 相对去年同期 |
| **毛利率指引** | 区间(如"65.5-67.5%") | 如有披露 |
| **EPS指引** | 区间或中点 | 如有披露 |
| **关键业务指引** | 如"AI芯片营收预期$10.7B" | 如有披露 |

#### 输出模板

```markdown
### Q2 2026指引

| 指引项目 | 数值 | 同比 | 环比 |
|---------|------|------|------|
```

### 4.3 目标价计算(10分钟)

#### 常用估值方法

**方法1: 前瞻P/E估值法**
```python
# 适用于盈利稳定增长的公司
target_price = FY_next_EPS × target_PE_multiple

# 示例(博通)
FY2027E_EPS = 15.80  # 基于分析师一致预期或自行预测
target_PE = 28.5  # 当前P/E或行业均值
target_price = 15.80 × 28.5 = $450
```

**方法2: DCF贴现现金流法**
```python
# 适用于现金流稳定的公司
# 步骤:
1. 预测未来5年自由现金流
2. 计算终值(Terminal Value)
3. 使用WACC贴现至现值
4. 除以总股本得到目标价
```

**方法3: EV/EBITDA估值法**
```python
# 适用于并购整合类公司
target_EV = FY_next_EBITDA × target_EV_EBITDA_multiple
target_equity_value = target_EV - net_debt
target_price = target_equity_value / shares_outstanding
```
