## 核心原则(Tier-0 规则)

### 专业术语规范
**本Skill统一使用以下专业术语**(必须严格遵守):
- **P/E**: 市盈率(Price-to-Earnings Ratio)
- **YoY**: 同比增长率(Year-over-Year)
- **QoQ**: 环比增长率(Quarter-over-Quarter)
- **CAGR**: 复合年增长率(Compound Annual Growth Rate)
- **FY**: 财年(Fiscal Year)
- **TTM**: 过去12个月滚动数据(Trailing Twelve Months)
- **EPS**: 每股收益(Earnings Per Share)
- **GAAP**: 美国公认会计准则
- **Non-GAAP**: 非公认会计准则(剔除一次性项目)

### 0. 呈现风格:少而精,不冗余

**黄金法则**: **不漏掉重要信息,但产出应该是精选而非凑数。"少而精"≠"信息删减",而是"高密度呈现"。**

**边界定义**:
- ✅ **必须保留**: 官方披露数据、管理层声明、市场核心关注点、投资决策关键数据
- ✅ **高密度呈现**: 每个数据点必须有洞察,每个洞察必须有数据支撑
- ❌ **必须删除**: 重复数据、显而易见的推导、过度解释、多余的过渡句

#### 呈现标准(对标台积电&博通财报风格)
| 维度 | 错误做法 | 正确做法 |
|------|---------|----------|
| **表格设计** | 列数>7列,信息冗余 | 核心指标3-5列,简洁清晰 |
| **文字密度** | 每个章节堆砌大段解释性文字 | 每个论点:数据→洞察,最多3-4句话 |
| **数据呈现** | 重复数据出现在多处 | 一个数据点只出现一次,在最合适的位置 |
| **章节数量** | 8-10个章节,过度拆分 | 最多7个章节,合并相关内容 |
| **emoji使用** | 每行都加emoji,视觉混乱 | 仅用于关键标识:✅(正确/优势)⚠️(风险/注意)❌(错误/禁止)💡(洞察) |
| **数据来源标注** | 每个表格下都重复标注完整来源 | 章节末统一标注,避免重复 |

#### 简洁度标准
```python
# 字数控制伪代码
max_words = {
    "核心财务数据章节": 200,      # 表格+1段💡洞察
    "历年营收章节": 300,          # 表格+3条关键洞察
    "业务拆解章节": 400,          # 表格+核心洞察+产品结构演变
    "管理层信息章节": 500,        # 核心亮点+Q2指引+全年展望
    "风险提示章节": 300,          # 风险矩阵表+敏感性分析
    "投资价值章节": 400,          # 核心优势+投资建议+目标价
}

# 总字数控制: 目标≤2500字(vs 当前3.8万字),信息密度提升15倍
```

#### 信息筛选原则
| 判断 | 标准 | 示例 |
|------|------|------|
| **必须保留** | 官方披露数据、管理层声明、市场核心关注点、投资决策关键数据 | "AI芯片营收$8.4B,+106% YoY"(官方数据+市场关注) |
| **可选保留** | 券商估计(需标注"估计")、历史案例(仅保留最相关1-2个)、敏感性分析 | "高盛预计FY27营收$115B(估计)"(需标注来源) |
| **必须删除** | 重复数据、显而易见推导、过度解释、多余过渡句 | "这说明公司业绩表现优异"(空洞表述) |

### 2. 财年(FY) vs TTM数据区分

| 数据类型 | 定义 | 适用场景 | 本Skill使用 |
|---------|------|---------|-----------|
| **财年(FY)** | 公司财年结束时的年度数据 | **历史对比、CAGR计算** | ✅ **必须使用** |
| **TTM** | 过去12个月滚动营收 | 实时追踪、当前估值 | ⚠️ 仅用于最新季度估值 |

**反面案例**: 博通数据验证中发现CompaniesMarketCap使用TTM数据导致6-15%误差

### 3. 三方交叉验证强制执行

**所有关键财务数据**(营收、净利润、EPS、毛利率)必须通过以下验证:

```
数据获取流程:
1. 从SEC/交易所获取官方数据 (主要来源)
2. 从公司IR新闻稿确认管理层声明 (二次确认)
3. 从至少2个权威第三方平台验证 (交叉验证)
4. 如有差异>1%,必须重新验证或标注差异原因
5. 最终数据必须注明所有来源
```

## 分析框架(必须完整执行)

### 阶段1: 数据收集与验证(强制执行)

#### 1.1 官方数据源定位

**美股公司**:
```
1. SEC EDGAR系统 → 搜索公司CIK编号
2. 获取最新Form 10-Q(季报)或10-K(年报)
3. 确认财报发布日期和财年周期
4. 提取Consolidated Statements of Operations
```

**港股/A股公司**:
```
1. 联交所/交易所官网 → 搜索公司代码
2. 获取最新季度/年度业绩公告
3. 确认财报发布日期和报告期
4. 提取综合损益表
```

**所有公司**:
```
5. 公司官方IR网站 → 查找Earnings Release/业绩公告
6. CEO/CFO官方声明确认关键数据
7. 如有财报会议,获取Management Guidance
```

#### 1.2 历年营收数据验证(10年)

**必须获取**: 最近10个财年的年度营收数据(如2016-2025)

**验证清单**:
- [ ] 所有数据来自Form 10-K(美股)或年度报告(其他市场)
- [ ] 财年截止日期明确标注
- [ ] 同比增长率计算验证
- [ ] CAGR(复合年增长率)计算验证
- [ ] 累计增长率计算验证
- [ ] 至少3个来源数据一致性验证

**输出格式**:
```markdown
## 历年营收趋势(2016-2025财年)

| 财年(FY) | 营收(美元) | 同比增长率 | 关键驱动因素 |
|---------|----------|----------|-------------|
| **2025** | **$XXX.XX B** | **+XX.X%** | 具体业务驱动 |
| ... | ... | ... | ... |

**关键洞察**:
- **2016-2025年CAGR**: XX.X%
- **累计增长**: +XXX.X% (从$XX.XB增至$XXX.XB)
- **增长最快三年**: YYYY年(+XX.X%,原因)、YYYY年(+XX.X%,原因)、YYYY年(+XX.X%,原因)
- **行业地位**: 具体排名或对比

> **数据来源**: SEC Form 10-K / 交易所公告  
> **交叉验证来源**: [具体平台名称]
```

#### 1.3 最新季度核心财务数据

**必须获取并验证**:
- [ ] 总营收(Revenue/Net Sales)
- [ ] 净利润(Net Income/Profit)
- [ ] EPS(每股收益,稀释)
- [ ] 毛利率(Gross Margin)
- [ ] 营业利益率(Operating Margin)
- [ ] 自由现金流(Free Cash Flow)
- [ ] 同比(YoY)和环比(QoQ)增长率

**计算验证**:
```python
# 同比增长率验证
yoy_growth = (current_quarter - same_quarter_last_year) / same_quarter_last_year * 100

# 环比增长率验证
qoq_growth = (current_quarter - previous_quarter) / previous_quarter * 100

# 必须与官方披露一致,误差≤0.1%
```

## 反面案例与教训(Anti-Patterns)

### ❌ 反面案例1: 数据来源不可靠

**错误示范**:
```markdown
## 历年营收趋势
| 年份 | 营收 |
|------|------|
| 2025 | $68.28 B |  ← ❌ 来自CompaniesMarketCap(使用TTM数据)
```

**正确做法**:
```markdown
## 历年营收趋势(2016-2025财年)
| 财年(FY) | 营收(美元) | 同比增长率 |
|---------|----------|----------|
| **2025** | **$63.89 B** | **+23.87%** |  ← ✅ 来自SEC Form 10-K
```

```python
    def cross_validate(self, data_sources: List[Dict]) -> bool:
        """交叉验证多个数据源"""
        if len(data_sources) < 3:
            raise ValueError("至少需要3个数据源进行交叉验证")
        
        # 验证数据一致性
```
