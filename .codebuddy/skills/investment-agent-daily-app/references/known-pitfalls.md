# 已知堵点与应对策略 — App版（v1.1）

> **用途**：投研鸭小程序数据生产过程中的已知堵点与应对方案。
> **核心原则**：先保真，再发布。核心行情缺失时宁可阻断，也不能用估算值、Mock值和新闻转述顶上。

---

## 数据采集堵点

| # | 堵点 | 降级路径 | 频率 |
|---|------|---------|------|
| 1 | Google Finance 403/超时 | → web_search → 东方财富/StockAnalysis | 高 |
| 2 | sparkline 历史数据无法获取 | → 基于当日价格 ± 波动估算（标记 `_sparklineEstimated: true`） | 高 |
| 3 | chartData 30天数据缺失 | → 用 7天sparkline + 线性插值扩展到 30 天 | 中 |
| 4 | metrics 数据缺失（PE/ROE等） | → web_search 补充 → 实在无法获取用 "—" 并标记 `_metricsMissing` | 中 |
| 5 | 某板块标的全部数据失败 | → 使用 stock-universe.md 中的备选标的替代 | 低 |
| 6 | 港股/A股数据获取困难 | → 东方财富/同花顺 → 智通财经 → web_search | 中 |
| 7 | 大宗期货 Google 不支持 | → OilPrice.com → 金投网 → web_search | 中 |

## JSON 生成堵点

| # | 堵点 | 解决方案 |
|---|------|---------|
| 1 | 某必填字段数据缺失 | 使用合理默认值并标记 `_fallback: true`，不留空 |
| 2 | sparkline 全为 0 或相同值 | 重新估算或用前日数据微调 |
| 3 | analysis 文本含 markdown | 运行文本清洗：去除 `**`、`|`、`>`、`- ` 等 |
| 4 | change 类型错误（string 而非 number） | 强制转换 `parseFloat(change)` |
| 5 | 枚举值越界 | 映射到最近的合法枚举值 |

## 上传堵点

| # | 堵点 | 解决方案 |
|---|------|---------|
| 1 | access_token 获取失败 | 检查 WX_APPID/WX_APPSECRET 配置 |
| 2 | 云数据库写入失败 | JSON 文件保留在本地，下次可手动重传 |
| 3 | JSON 文件含特殊字符导致 query 解析失败 | upload_to_cloud.py 已内置 `_sanitize_for_query` 清洗 |
| 4 | 部分集合上传成功/部分失败 | 成功的数据生效，失败的保持旧数据 |

## 小程序端兼容性堵点

| # | 堵点 | 解决方案 |
|---|------|---------|
| 1 | sparkline 缺失导致 mini-chart 空白 | 小程序端 api.js 有 Mock 数据兜底 |
| 2 | 板块标的为空数组 | 小程序端显示"暂无数据" |
| 3 | 红绿灯不足7项 | 小程序端按实际数组长度渲染 |
| 4 | 新增字段小程序未适配 | 新字段向后兼容，小程序忽略未识别字段 |

---

## 端到端零堵点保障

1. 数据源异常 → 自动切换备选源
2. sparkline 缺失 → 自动估算生成
3. 板块标的不足 → 自动使用备选标的
4. 某个JSON完全无法生成 → 标记 `_fallback: true`，不上传该集合
5. 上传失败 → JSON 文件保留，不影响其他集合
6. **任何环节不停下等用户**
